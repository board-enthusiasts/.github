# Board Enthusiasts Support

This document explains where to go for help, feedback, and community interaction across Board Enthusiasts (BE) spaces.

## Quick links

- Website: [boardenthusiasts.com](https://boardenthusiasts.com/)
- Community Discord: _https://discord.gg/cz2zReWqcA_
- General inquiries: [contact@boardenthusiasts.com](mailto:contact@boardenthusiasts.com)
- Technical issues: [support@boardenthusiasts.com](mailto:support@boardenthusiasts.com)

## Where should I go?

### I want to learn what Board Enthusiasts is
Start with the website:

- [boardenthusiasts.com](https://boardenthusiasts.com/)

The website is the main public-facing entry point for BE.

### I want to talk with the community
Use the BE Discord.

Discord is the best place for:
- community discussion
- general help
- early feedback
- questions from players
- informal conversation around BE offerings

### I want to report a bug or request a feature
Use the BE website or Discord as the user-facing entry point.

Maintainers may convert actionable feedback into GitHub issues and project items for implementation tracking.

GitHub is the canonical place where BE maintainers track actionable engineering work, but it is not always the preferred first stop for general users.

### I want to contribute code or documentation
Use GitHub.

Please review:
- this repository’s [README](README.md)
- the [contribution guidelines](CONTRIBUTING.md)
- any repository-specific documentation

### I need help with a specific repository
First check:
- the repository README
- the docs for that repository
- existing issues or discussions if enabled

If you are still unsure, use the BE Discord unless the repository explicitly asks you to use a different workflow.

## Support model

BE uses different spaces for different purposes:

- **Website** for public-facing discovery and entry points
- **Discord** for community discussion and general help
- **GitHub** for code, contributor workflows, and actionable implementation tracking

Not every question or piece of feedback should begin as a GitHub issue.

## Before opening GitHub issues

Please do not open a GitHub issue for:
- general community discussion
- vague or early-stage feedback that is not yet actionable
- routine support questions already better suited for Discord
- complaints without enough detail for maintainers to act on

Maintainers may close, relabel, defer, or redirect issues that are better handled through another channel.

## Security issues

Do **not** report security vulnerabilities in public issues.

If this repository or organization provides a `SECURITY.md`, follow the instructions there.

## Conduct expectations

Please review the [code of conduct](CODE_OF_CONDUCT.md); engage respectfully and constructively in all BE spaces.
